# 記事を Wix まで仕上げる（Cursor ディレクション）

**Claude Code で下書きができたあと**、この手順に沿って進めると、画像の保存・log/stock 更新・Wix 投稿まで迷わず完了できます。  
各ステップで「誰が・何をするか」を Cursor が案内します。

---

## 前提

- 記事の**本文**は Claude Code が書いた（OK を出した）
- Claude Code が**画像用プロンプト**も出してくれた

---

## ステップ1：あなた → ChatGPT で画像を生成

1. Claude Code が書いた**画像生成用プロンプト**をコピーする。
2. **ChatGPT** を開き、そのプロンプトを貼り付ける。
3. 「このプロンプトで画像を生成して」と送る。
4. 生成された画像を**ダウンロード**する。

→ ここまでできたら **ステップ2** へ。

---

## ステップ2：あなた → 画像を共通の場所に保存

1. ダウンロードした画像ファイルを、次のフォルダに保存する：
   - **`matchmaking-blog-workflow/drafts/images/`**
2. ファイル名の例：**`YYYY-MM-DD_eyecatch.png`**（例：`2026-03-07_eyecatch.png`）
   - 図解が複数ある場合は `2026-03-07_diagram_〇〇.png` など。

→ 保存できたら **ステップ3** へ。

---

## ステップ3：あなた → Cursor に仕上げを依頼

**Cursor のチャット**で、次のように送る（日付・タイトル・トピック名は実際のものに置き換える）：

```
この記事の本文を drafts/ に draft_YYYY-MM-DD_タイトル.md で保存して。
Wix用の記事全文と、GBP用の概略テキストを drafts/ に保存して。
使ったトピック「〇〇」を used/log.md に日付付きで記録して、stock.md からは削除して。
```

※ すでに drafts に保存済みの場合は「used/log に記録して、stock から削除して、GBP 用テキストを作って」だけでも可。

→ Cursor がファイル保存・log 更新・stock 削除・GBP 用テキストを作成したら **ステップ4** へ。

---

## ステップ4：あなた → Wix に投稿

1. **Wix** のブログ管理画面を開く。
2. **新規投稿**（または下書き）を作成する。
3. **本文**：`drafts/draft_YYYY-MM-DD_タイトル.md` の内容をコピーして Wix の本文欄に貼り付ける。
4. **画像**：`drafts/images/YYYY-MM-DD_eyecatch.png` を Wix のメディアにアップロードし、記事のアイキャッチ（または該当位置）に挿入する。
5. 必要ならタイトル・カテゴリを設定して**公開**（または下書き保存）。

→ ここまでで **記事の仕上げは完了**です。

---

## ステップ5（任意）：GBP に投稿

記事を Wix で公開したあと、**Google Business Profile** に投稿する場合：

1. `drafts/gbp_YYYY-MM-DD.txt` を開く。
2. 内容をコピーして GBP の投稿欄に貼り付ける。
3. 詳細リンクに **Wix のブログ記事 URL** を設定する。

---

## まとめ（誰が何をするか）

| ステップ | 担当 | やること |
|----------|------|----------|
| 1 | あなた | ChatGPT にプロンプトを渡して画像を生成・ダウンロード |
| 2 | あなた | 画像を `drafts/images/` に保存 |
| 3 | **Cursor** | 記事を drafts に保存・log/stock 更新・GBP 用テキスト作成 |
| 4 | あなた | Wix に本文と画像を投稿 |
| 5 | あなた | （任意）GBP に投稿 |

**このファイルを開いた状態で、上から順に進めれば、Cursor がディレクションしているように進められます。**
