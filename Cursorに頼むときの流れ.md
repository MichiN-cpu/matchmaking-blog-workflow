# Cursor に頼むときの流れ（あなたが最小限の指示で済むように）

あなたの希望：「私が Cursor に記事の種を言う → Cursor が Claude Code を呼んで下書き → 私が OK → Cursor が ChatGPT を呼んで画像を作って保存 → その画像を使って記事を Wix に保存」

**技術的な制約**（いまの環境でできること・できないこと）を整理したうえで、**Cursor にどう頼めば近い流れになるか**を書きます。

---

## できること・できないこと

| あなたの希望 | いまできる？ | 説明 |
|--------------|--------------|------|
| Cursor が **Claude Code を呼び出す** | ❌ 不可 | Cursor と Claude Code は別アプリで、Cursor から Claude Code を起動したり依頼を送ったりする機能はありません。 |
| Cursor が **下書きを書く** | ✅ 可 | あなたが「このトピックで下書き書いて」と頼めば、Cursor が BLOG_POLICY 等を読んで**同じ方針で**下書きを書きます（Claude Code の代わりに Cursor が書く形）。 |
| あなたが **OK を出す** | ✅ 可 | 下書きを読んで「OK」か「ここを直して」と Cursor に伝えるだけです。 |
| Cursor が **画像を作って保存する** | ✅ 可 | Cursor には画像生成の機能があります。あなたが「OK。画像作って drafts/images に保存して」と頼めば、**画像用プロンプトを考えて画像を生成し、drafts/images/ に保存**できます（ChatGPT は使わず Cursor が生成）。 |
| Cursor が **log/stock 更新・GBP 用テキスト作成** | ✅ 可 | いつもどおり Cursor に依頼すれば対応できます。 |
| Cursor が **Wix に直接アップロードする** | ❌ 不可 | Wix は外部のウェブサービスなので、Cursor から直接ログインして投稿する機能はありません。**Wix 用の本文と画像を drafts/ に用意する**までが Cursor の役割で、**実際の Wix への投稿はあなたがブラウザで行う**形になります。 |

---

## Cursor に頼むときの流れ（現実的な案）

あなたがするのは「**トピックを言う → 下書きを読んで OK を言う → 一言で画像・保存を頼む → 最後に Wix に貼る**」だけにできます。

### ステップ1：あなた → Cursor に記事の種を伝える

**Cursor のチャット**で、例えばこう送る：

```
記事の種は「恋愛・結婚がうまくいかないときの5つのパターン」。
BLOG_POLICY_ASUNARU.md と ASUNARU_EHIME_BRIEF.md と EDITORIAL_GUIDELINES.md を読んで、
このトピックでブログの下書きを書いて。出したら一度止まって、OK かどうか確認させて。
```

→ **Cursor** が 3 ファイルを参照して下書きを書き、本文を返します。  
（Claude Code は呼び出せないので、**Cursor が同じ方針で下書きを書く**形になります。）

---

### ステップ2：あなたが OK を出す

内容を読んで、

- **OK のとき**：「OK。この記事で進めて」
- **直したいとき**：「〇〇を直して」と具体的に伝える

→ 直しがあれば Cursor が修正して再度出します。

---

### ステップ3：あなた → Cursor に画像・保存を一括で頼む

**OK が出たら**、次のように**一言**で頼む：

```
OK。この記事用のアイキャッチ画像を生成して drafts/images/ に保存して。
あと、この記事を drafts/ に保存して、used/log に記録、stock からこのトピックを削除して。GBP 用テキストも作って。
```

→ **Cursor** が次のことをします：

1. 記事に合う**画像用プロンプト**を考え、**画像を生成**して **drafts/images/YYYY-MM-DD_eyecatch.png** に保存  
2. 記事本文を **drafts/draft_YYYY-MM-DD_タイトル.md** に保存  
3. **used/log.md** に日付とトピックを記録  
4. **topics/stock.md** からそのトピックを削除  
5. **drafts/gbp_YYYY-MM-DD.txt** に GBP 用テキストを作成  

（画像は Cursor の画像生成で作成するため、**ChatGPT は使いません**。同じように「記事に合う画像」を Cursor が作って保存します。）

---

### ステップ4：あなた → Wix に投稿する（ここだけ手動）

**Wix** は Cursor から操作できないため、**あなたがブラウザで**次を行います：

1. **drafts/draft_YYYY-MM-DD_タイトル.md** の本文を Wix の記事編集画面に貼り付ける  
2. **drafts/images/YYYY-MM-DD_eyecatch.png** を Wix のメディアにアップロードして、記事のアイキャッチ（または本文中）に挿入する  

→ ここまでで「記事を Wix に保存（公開）」が完了です。

---

## まとめ：あなたが Cursor に頼むときの「言い方」

1. **下書きが欲しいとき**  
   「記事の種は〇〇。BLOG_POLICY と ASUNARU_EHIME_BRIEF と EDITORIAL_GUIDELINES を読んで下書き書いて。出したら止まって確認させて。」

2. **OK して、画像・保存まで一気にやりたいとき**  
   「OK。この記事用のアイキャッチ画像を生成して drafts/images に保存して。この記事を drafts に保存して、used/log に記録、stock から削除、GBP 用テキストも作って。」

3. **Wix に載せるとき**  
   あなたが drafts/ の本文と drafts/images/ の画像を Wix に貼り付け・アップロードする。

---

## Claude Code や ChatGPT を必ず使いたい場合

- **Claude Code に書いてもらいたい** … Cursor からは呼べないので、**あなたが** Claude Code を開き、Cursor が作った「Claude Code 用依頼文」を貼って依頼する形になります。  
- **ChatGPT に画像を生成してほしい** … その場合は、従来どおり「Claude Code が書いた画像用プロンプトをあなたが ChatGPT に貼る → 画像をダウンロードして drafts/images に保存」の手順になります。Cursor は「画像用プロンプトを出す」「保存場所の案内」までです。

「できるだけあなたの手を省きたい」場合は、上記の **Cursor に一括で頼む流れ**（下書きも画像も Cursor が担当）がいちばんシンプルです。
