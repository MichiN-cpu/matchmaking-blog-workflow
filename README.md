# 結婚相談所ブログ運用ワークフロー

## 概要
- **トピックストック**: ブログ記事の種を `topics/` にストック
- **日次運用**: 毎日1トピックを選び、記事下書きを作成（必要なら図解も）→ 画像生成 → **内容確認** → Wix下書き用テキストを出力
- **品質**: 顧客目線・失敗しうる点・会員の幸せな未来に繋がるか等を `EDITORIAL_GUIDELINES.md` でチェック
- **マルチAI**: Cursor / Claude Code で指示を組み立て、Gemini 等にも依頼して下書き完成まで連携可能（→ `SETUP.md`）
- **GBP連携**: 記事公開後、Google Business Profile（GBP）に概略＋詳細リンク用のテキストを出力

## フォルダ構成
```
matchmaking-blog-workflow/
├── README.md              # このファイル
├── トピックストック.md     # 記事の種をすぐ開く用（→ topics/stock.md への案内）
├── SETUP.md               # Claude Code 拡張・マルチAI（Gemini等）のセットアップ
├── WORKFLOW.md            # 日次ブログ作成の手順（依頼文例つき）
├── EDITORIAL_GUIDELINES.md # 編集方針・品質チェック・図解の要不要
├── topics/
│   └── stock.md           # 記事の種ストック（ここに追記）
├── drafts/                # 下書き（Wix用・GBP用）
│   ├── images/            # アイキャッチ・図解画像
│   └── diagrams/          # 図解の構成案・メモ
└── used/
    └── log.md             # 使用済みトピックログ
```

## トピックストックを常に見えるようにする

- **ルートの「トピックストック.md」** … プロジェクトを開いたときに目につく場所にあります。開くと本番のストック（`topics/stock.md`）へのリンクと、開き方の案内があります。
- **ピン留め** … `topics/stock.md` を開き、タブを右クリック → **「ピン留め」**にすると、他のファイルを開いてもタブが残り、いつでもワンクリックで開けます。
- **クイックオープン** … `Cmd + P`（Mac）/ `Ctrl + P`（Win）→「stock」と入力 → Enter で `topics/stock.md` を開けます。

## 使い方

### 1. トピックをストックする
`topics/stock.md` に、書きたい記事のタイトルやキーワードを1行ずつ（またはリストで）追加する。ルートの **トピックストック.md** からもリンクで開ける。

### 2. 毎日のブログ作成フロー
1. **今日のトピック選択**  
   `topics/stock.md` から1つ選ぶ（先頭から順や、日付でローテーション）。
2. **記事作成**  
   Cursor / **Claude Code** に依頼。Gemini 等へ指示を回す場合は `SETUP.md` 参照。  
   `EDITORIAL_GUIDELINES.md` に沿って、顧客目線・失敗点・多角的観点・必要なら図解構成も含めて作成。
3. **品質チェック**  
   `EDITORIAL_GUIDELINES.md` の客観視・共感・失敗点・現実性・多角的観点でチェック。不足があれば修正。
4. **あなたの確認**  
   内容を確認し、「OK」なら次へ、「修正して」なら指示を伝える。
5. **画像・図解**  
   アイキャッチや図解が必要なら生成し、`drafts/images/` 等に保存。
6. **Wix下書き用テキスト出力**  
   記事本文＋画像配置メモを `drafts/` に保存。Wixの記事下書きにコピー＆ペースト。
7. **使用済み記録**  
   使ったトピックを `used/log.md` に日付とともに記録。

### 3. GBP用テキスト
記事をWixで公開したら、`drafts/` に保存した「GBP用概略」をコピーし、Google Business Profile の「投稿」や「アップデート」に貼り付ける。詳細はWixブログへのリンクで誘導。

- **品質・図解のルール**: `EDITORIAL_GUIDELINES.md`

## 注意
- Wix・GBPへの実際の保存・投稿は、手動で行うか、各サービスのAPI連携を別途検討してください。
- 画像は `drafts/images/`、図解の構成案は `drafts/diagrams/` に保存し、Wixのメディアからアップロードして記事に挿入します。
