# ブログ記事 編集方針・品質チェック

結婚相談所ブログは「会員が増え、入会した会員が婚活を契機により幸せな反応パターンで、幸せな家族と過ごす未来」に繋がることを目指します。記事作成・チェック時は以下を参照してください。

---

## 1. 客観視チェック（ゴール整合）

- [ ] **会員の幸せな未来に繋がるか**  
  記事の内容が、読者（会員・見込み会員）の「婚活→成婚→家族の幸せ」という流れに現実的に貢献しているか。
- [ ] **結婚相談所の役割と限界**  
  相談所としてできること・できないことを過不足なく伝え、過大な期待や誤解を招いていないか。
- [ ] **倫理的な表現**  
  押し売り感や不安をあおる表現がなく、選択の自由と多様な幸せを尊重しているか。

---

## 2. 顧客目線・共感

- [ ] **読者の立場で書けているか**  
  「相談所目線」ではなく、婚活中の人・迷っている人の気持ちに立った表現か。
- [ ] **共感できる具体性**  
  抽象論だけでなく、「こんなときありませんか？」といった具体的なシーンや悩みに触れているか。
- [ ] **トーン**  
  上から目線ではなく、寄り添う・一緒に考えるトーンになっているか。

---

## 3. 失敗しうる点の視点

- [ ] **どこで失敗しやすいか**  
  テーマに応じて「こうするとうまくいきにくい」「よくある落とし穴」を押さえているか。
- [ ] **リスクの伝え方**  
  不安をあおらず、現実的な注意点として書けているか（例：プロフィールの書き方、見学の準備、期待値の持ち方など）。
- [ ] **フォールバック**  
  うまくいかないときの考え方や、相談所・他サービスへの相談など、次の一歩が示されているか。

---

## 4. 会員にとって現実的か

- [ ] **時間・コスト・環境**  
  読者の生活実態（仕事、子育て、予算など）を踏まえた現実的なアドバイスか。
- [ ] **段階的な一歩**  
  いきなり大きな決断ではなく、「まずここから」という取り組みやすい一歩があるか。
- [ ] **多様な状況**  
  再婚、地域、年齢、働き方など、いろいろな会員像を想定したときにも無理のない内容か。

---

## 5. 多角的な観点（心理学以外も含める）

記事のテーマに応じて、次の観点を必要に応じて盛り込む。

| 観点 | 例（記事で扱いうる内容） |
|------|---------------------------|
| **心理学** | 恋愛・結婚への態度、コミュニケーション、ストレス、動機づけ |
| **社会学** | 結婚・家族の社会規範の変化、地域・コミュニティ、ライフコースの多様化 |
| **歴史・文化** | お見合い・結婚相談の歴史、文化差、慣習の変化 |
| **遺伝学・生物学** | 触れ方に注意しつつ、進化心理学や親密関係の研究の「考え方」の紹介など |
| **マーケティング** | 自分を伝える（プロフィール・自己紹介）、ニーズとマッチング、情報の非対称性 |
| **ライバル・選択肢** | 婚活アプリ・街コン・紹介など「結婚・恋愛の他の選択肢」との違いや、併用・使い分け |
| **法律** | 入籍・婚姻届、氏（別姓・夫婦別姓）、遺産相続・遺言、養子縁組、再婚時の親権・姓、婚姻前の契約や財産など。成婚後のトラブル予防のため「専門家に相談を」と促す表現を入れる |

※ 専門用語は必要最小限にし、読者が「なるほど」と思えるレベルでかみ砕いて書く。  
※ **法律的観点**：遺産相続・入籍のタイミング・別姓（夫婦別姓）・養子縁組・再婚時の姓や親権など、結婚に伴い起こりうる法的な問題に触れる記事では、断定せず「弁護士・司法書士・行政書士など専門家への相談を検討してください」と案内する。

---

## 6. 図解の要不要

- **図解を入れる目安**  
  手順・流れ・比較・選択肢・段階など、「図にすると理解しやすい」テーマでは図解を検討する。
- **例**  
  見学の流れ、結婚相談所とアプリの比較、成婚までのステップ、プロフィール作成の手順 など。
- 図解が必要な記事では、`drafts/diagrams/` に図の案（説明文や構成メモ）を保存し、必要に応じて画像を生成して Wix に配置する。

---

## 記事作成・レビュー時の依頼文例（AI向け）

記事を書く・直すときに、AI（Cursor / Claude Code / Gemini 等）へ渡すときの例：

```
この記事について、EDITORIAL_GUIDELINES.md の
1. 客観視チェック（会員の幸せな未来に繋がるか）
2. 顧客目線・共感
3. 失敗しうる点
4. 会員にとって現実的か
5. 多角的な観点（心理学以外に社会学・歴史・マーケ・ライバル視点・法律的観点：入籍・別姓・相続・養子縁組など）
に沿ってチェックしてほしい。不足があれば指摘と修正案を出して。
```

図解が必要な場合の例：

```
この記事は [見学の流れ / 比較表 / 成婚ステップ など] を説明しているので、
EDITORIAL_GUIDELINES.md の「図解の要不要」に従い、
図解の構成案（何をどの順で示すか）を出して。
必要なら drafts/diagrams/ にメモを保存して。
```
